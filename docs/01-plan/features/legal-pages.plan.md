## Executive Summary

| Perspective | Content |
|---|---|
| **Problem** | 회원가입·콘텐츠 생성·SNS 게시 기능을 운영하면서 사용자에게 약관·개인정보 처리 정책을 명시하지 않음. 법적 의무 미충족 + 사용자 신뢰 부족 |

| **Core Value** | 법적 의무 충족 + 사용자에게 "이 서비스가 내 데이터를 어떻게 다루는지" 투명하게 공개 |

## 1. Goals

- 회원가입·콘텐츠 게시 시 사용자가 동의 가능한 약관·방침 페이지 공개
- 한국 개인정보보호법 필수 고지 항목 누락 없이 포함
- 운영 주체("주식회사 비누컴퍼니")와 연락 채널 명시
- footer 어디서나 1클릭 접근

## 2. Non-Goals (1차)

- 약관 동의 체크박스를 회원가입·온보딩 흐름에 강제 삽입 (2차)
- 약관 버전 관리·이력 추적 (2차)
- 동의 철회 워크플로(회원 탈퇴 + 데이터 삭제) (2차)
- 영문/일문 등 다국어 (출시 후)
- 별도 쿠키 정책 페이지 — 현재 1st-party 쿠키만 사용해서 본문에 통합

## 3. 핵심 결정 사항

| # | 결정 | 채택 | 이유 |
|---|---|---|---|
| 1 | 운영 주체 | **주식회사 비누컴퍼니** | 사용자 결정 |
| 2 | 14세 미만 가입 | **불가** | 보호자 동의 절차 부담 회피, 해외 서비스도 다수 채택 |
| 3 | 생성 곡 저작권 | **사용자 귀속 + 우리에게 서비스 운영·개선·프로모션 이용권** | Suno·Udio 표준. MiniMax API 의존이라 파생권만큼은 우리 이용권 명시 필요 |
| 4 | URL 구조 | `/terms`, `/privacy` | 짧고 표준적, SEO 친화 |
| 5 | 레이아웃 | `(legal)` route group + 미니멀 shell (로고 + 홈 링크) | (main) shell 분리 — 비가입자도 부담 없이 열람 |
| 6 | 톤 | 한국어 평어체, 섹션·소제목 명확 | MONO 다른 UI와 일관(반말 X, 친근 존댓말) |
| 7 | 시행일 | 문서 상단에 명시(`2026-05-22`) + 변경 시 갱신 | 한국 법 필수 |

## 4. 페이지 구조

### `/terms` 이용약관 (12개 조항)

1. 목적
2. 정의 (서비스, 회원, 콘텐츠, 크레딧)
3. 약관의 효력 및 변경
4. 회원가입 (14세 이상, OAuth 사용, 자격)
5. 회원 정보 변경 (아이디 1회 제한, 이름 14일 2회 등)
6. 서비스 이용 (무료 크레딧 정책, 모델 제한, 잠긴 기능)
7. 콘텐츠의 권리 (저작권 사용자 귀속 + 서비스 이용권)
8. 금지 행위 (저작권 침해, 음란·폭력, 어뷰징, 크롤링)
9. 게시물 관리 (신고·삭제 권한)
10. 책임 제한 (AI 생성 결과의 한계, 외부 서비스 장애)
11. 분쟁 해결 (관할 법원: 서울중앙지방법원)
12. 부칙 (시행일)

### `/privacy` 개인정보처리방침 (10개 조항)

1. 개인정보 수집 항목
   - **필수**: 이메일, OAuth 식별자, username, 가입 일시
   - **선택**: display_name, bio, avatar/cover, SNS 링크
   - **자동 수집**: 생성 곡(prompt·가사·오디오·커버), 좋아요, 컬렉션, 재생 기록, 일일 크레딧 사용량, 접속 IP/UA
2. 수집 목적 (서비스 제공, 인증, 개인화, 부정 이용 방지, 통계)
3. 보유·이용 기간 (회원 탈퇴 시까지 + 법령 보존 기간)
4. **제3자 제공** (현재 없음)
5. **처리위탁**
   - Supabase (DB·Storage·Auth) — 호스팅: AWS (서울 또는 도쿄 리전)
   - MiniMax AI (음악·이미지·가사 생성) — 호스팅: 해외
   - Google OAuth, Kakao OAuth — 인증 위임
6. 이용자의 권리 (열람·정정·삭제·처리 정지·동의 철회)
7. 파기 절차·방법
8. 안전성 확보 조치 (전송 암호화, RLS, 비밀번호 OAuth 위임)
9. **개인정보 보호 책임자**
   - 이름·소속: TBD (개발팀 운영자)
   - 연락처: contact@... (운영 주체 결정 후 확정)
10. 방침 변경 고지 (사전 공지 + 시행일 명시)

## 5. 라우팅 + 파일 구조

```
app/
├── (legal)/
│   ├── layout.tsx        — 미니멀 shell: 헤더(로고만) + 본문
│   ├── terms/page.tsx
│   └── privacy/page.tsx
└── (main)/layout.tsx     — footer 링크 href 업데이트
```

- `(legal)` route group으로 shell 분리 (사이드바·미니바·인증 없이도 열람)
- 본문: max-width 720px 중앙 정렬, h1·h2·p 명확한 위계, 본문 가독성 우선
- 푸터: 시행일·운영 주체 노출

## 6. 카피·법적 표현 톤

- "~합니다" 평어체 존댓말 (반말 X)
- 정의는 따옴표로 명시: 회원(이하 "회원")
- 조항 번호 + 제목 구조
- 사용자 권리 부분은 능동형: "회원은 ~를 요청할 수 있습니다"
- 면책 표현은 표준화: "~로 인해 발생한 손해에 대해 책임을 지지 않습니다"
- 외부 서비스명은 영문 그대로 (Supabase, MiniMax, Google, Kakao)
