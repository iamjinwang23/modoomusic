## 결론 먼저

> **"AI로 만든 내 노래가 바로 콘텐츠가 되는 플랫폼"**

Suno와의 차이는 생성 품질이 아니라 **만들고 난 다음의 경험 전체를 소유**하는 것.

```
만들기 → 감상 (카라오케) → 숏폼 영상 → 공유 → 유입
    ↑                                              ↓
    └──────────── 팔로우 / 커버 / 챌린지 ──────────┘
```

## 탐색 과정

### 포지션 정의
- **비전**: B — "모두가 함께 즐기는 음악" (플랫폼). 창작물이 공유·소비되는 커뮤니티.
- **타깃**: A(일반인)가 주 대상이지만, B(크리에이터)·C(K-팝 팬)이 활성화되길 원함.

### 경쟁사 분석
| 서비스 | 강점 | 약점 |
|---|---|---|
| Suno | 생성 품질, 레퍼런스 오디오 커버, 커뮤니티(팔로우·좋아요·리믹스) | 영어 중심, 숏폼 영상 출력 없음, 가사 싱크 없음 |
| MiniMax | API 공급자, 다양한 모델 | 소비자 앱 없음 |
| Mureka | 가사 하이라이팅 (현재 재생 줄 강조) | — |

**교훈**: Suno는 이미 커뮤니티 기능도 잘 갖춤. 생성 품질·커뮤니티 기본기로는 따라잡기 어려움.

### 하지 않을 것 (명시적 제외)
- **스트리밍 유통** (멜론·벅스·지니) — 저작권 고정비 감당 불가. TJ·한저협 실패 전례 있음.
- 생성 품질 정면 경쟁
- 과도한 라이선싱 딜

### 차별화 포인트 (결정)
1. **가사 싱크 (카라오케 스타일)** — 인앱 감상 경험 강화, Whisper 기반
2. **가사 오버레이 MP4 내보내기** — 숏폼 콘텐츠 즉시 생성, TikTok/Reels 바이럴 루프
3. **가격 경쟁력** — 전략 레벨 결정 필요
4. **한국 팬 문화** — 응원가, 커버, 스타일 참조 (Music 2.6 이미 구현)

## 기술 참조 — 가사 싱크 구현

### 3계층 구조
1. **타임스탬프 생성** — Whisper word-level timestamp (한국어 잘 작동). 생성 후 서버사이드 실행.
2. **저장 포맷** — LRC 또는 JSON `{line, start, end}`
3. **프론트 렌더링**
   - `currentTime` 폴링 (requestAnimationFrame 또는 timeupdate)
   - 현재 줄 `active` 클래스 토글
   - `scrollIntoView({block: 'center', behavior: 'smooth'})`
   - 글자 단위 채우기: `clip-path: inset(0 X% 0 0)`, 진행도 = `(currentTime - lineStart) / (lineEnd - lineStart)`
