# 앱 ↔ 웹 UI/UX 파리티 감사 리포트 (2026-07-08)

> **원칙:** 웹(modoonorae.com)이 디자인 source of truth. 앱은 웹에 맞춘다. (플랫폼상 불가피한 모바일 적응만 예외 — 데스크톱 사이드바→하단탭, 섹션 스택→필터칩 등. 단 시각 언어는 웹과 일치.)
>
> **감사 방법:** iOS 시뮬레이터(iPhone 17 Pro) 앱 화면 스크린샷 + 웹(로그인 상태) 브라우저 캡처를 화면별 대조.
> **감사 범위:** 로그인·둘러보기·커뮤니티·라이브러리·만들기·플레이어·프로필. (2차: 설정·검색·알림·커뮤니티 상세·게시글 상세 — 수정하며 확인)

| ID | 항목 | 현재(앱) | 목표(웹 기준) |
|---|---|---|---|

| **G4** | **크리에이터 아바타 표시** | 플레이어/곡 컨텍스트에 아티스트 **이름만**, 아바타 없음 | 크리에이터 **아바타 추가**(웹은 곡·프로필에 아바타 노출) |
| **G5** | **아티스트명 색 = 회색(secondary)** | 목록=회색, 플레이어=바이올렛(불일치) | **회색으로 통일** |

## 1. 만들기 (Create) — **최우선 / 최대 작업**

> 유저 강조: "만들기는 특히 웹을 많이 참조. 기능·UI 누락 다수." → 사실상 웹 스펙대로 재구성.

### 웹 만들기 스펙 (앱이 갖춰야 할 전체)
**공통 상단:** `✦크레딧 잔액` · **심플/고급 토글**
- **심플 모드:** `설명` textarea("만들고 싶은 노래를 자유롭게…한 번에 완성") + 인스트루멘탈 토글 + **`+가사`** 버튼 + `음악 만들기 ✦비용`
- **고급 모드:**
  - `가사` textarea + 인스트루멘탈 토글 + **`✦AI 가사`** 버튼 + 확대(fullscreen)
  - `스타일` textarea + **퀵칩**(🔄셔플 · 바이올린 · 잔잔한 · R&B · 트럼펫 · K-pop · ▸더보기)
  - `곡 제목(선택)` · **`보컬 성별` 여성/남성** · 모델 드롭다운
  - `음악 만들기 ✦비용` CTA

### 앱 누락 (수정)
| ID | 누락 |
|---|---|
| M1 | **심플/고급 모드 토글** 자체 없음 |
| M2 | **스타일 퀵칩**(셔플·바이올린·잔잔한·R&B·트럼펫·K-pop·더보기) |
| M3 | **보컬 성별(여성/남성)** |
| M4 | **크레딧 잔액(✦N)** 표시 |
| M5 | **AI 가사 생성 버튼**(심플=+가사, 고급=✦AI가사) |
| M6 | **가사 확대(fullscreen)** |
| M7 | 모델 표기 통일 — 웹 드롭다운(v2.x) vs 앱 카드(Music 2.6/2.5+). 라인업·명명 일치 |

## 수정 우선순위(제안)

1. **공유 컴포넌트 먼저**(G1 커버세로 · G2 통계 · G3 워딩 · G4 아바타 · G5 색 · H1 헤더) → 여러 화면 동시 반영.
2. **만들기 재구성**(M1~M7) — 최대 작업, 웹 스펙 밀착.
3. **프로필 그리드**(PR1~PR2) + **커뮤니티 재구성**(C1).
4. **라이브러리**(LIB1~LIB7) · **플레이어**(P2·P4·P5·P6·P8) · **로그인**(L1·L2).
5. 2차 모달 감사(설정·검색·알림·커뮤니티상세·게시글상세)는 수정 라운드에서 확인.
